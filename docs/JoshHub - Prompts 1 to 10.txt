Architecture (buildable + future-proof)
Recommended stack (because you’re already in this world)
                        * Next.js (App Router) + TypeScript

                        * Tailwind + shadcn/ui (clean UI fast)

                        * Vercel deploy (one click, stable on mobile + PC)

App structure (practical)
                           * src/app/ routes

                           * src/data/apps.ts (single source of truth for your catalogue)

                           * src/components/ UI building blocks

                           * src/lib/ helpers (search, tags, local storage)

                           * src/content/ markdown for personal pages (“Faith”, “Family”, etc.)

Data model (start dead simple)
For each “thing” (app/game/page), store:
                              * id, name, type (app/game/doc), category

                              * urls[] (because many have multiple versions)

                              * status (ok/broken/wip)

                              * tags[] (dubbo, dcs, game, react, etc.)

                              * notes, lastTouched

Storage strategy (phased)
Phase 1 (MVP): static config + local-only history
                                 * Everything public is just links + markdown content.

                                 * “Recently used” stored in localStorage.

Phase 2 (Private features): add optional login + sync
Pick one:
                                    * Supabase (free tier, easiest auth+db)

                                    * OR local-first encrypted vault (IndexedDB + WebCrypto), with manual export/import for syncing.

I’d do Supabase only when you’re ready to store genuinely private stuff.

Embedding vs linking (important detail)
Some sites/apps won’t allow iframes (security headers), especially itch.io sometimes.
So JoshHub should:
                                       * Default: open in new tab

                                       * Optional: “Try embed view” toggle (iframe) and gracefully fall back if blocked.
